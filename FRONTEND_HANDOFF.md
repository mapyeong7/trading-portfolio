# 주식 수익률 대회 사이트 프론트엔드 인계 문서

## 1. 서비스 목적

이 서비스는 여러 참가자가 매월 한 종목을 제출하고, 해당 종목의 수익률로 월간 순위와 누적 순위를 확인하는 주식 수익률 대회 사이트입니다.

핵심 의도는 다음과 같습니다.

- 일반 사용자는 로그인 없이 대회 현황과 순위를 쉽게 확인한다.
- 관리자는 로그인 후 참가자, 기준월, 참가 종목, 매수/매도 정보, 메모, 결과 확정을 관리한다.
- 결과가 확정된 월간 대회 데이터는 나중에 가격이나 상태가 바뀌어도 보존된다.
- 결과 확정 전에는 현재가 기준으로 임시 순위를 보여준다.
- 결과 확정 후에는 확정일 기준 종가로 공식 순위를 고정한다.

## 2. 현재 기술 스택

- Frontend: React + Vite + TypeScript
- Backend: Cloudflare Worker
- Database: Cloudflare D1
- Deploy: Cloudflare Workers
- Price Source: Worker에서 외부 시세 데이터를 조회
- Auth: 관리자 계정 DB 기반 로그인

## 3. 사용자 구조

### 공개 사용자

로그인 없이 접근합니다.

할 수 있는 일:

- 대시보드 확인
- 월간 TOP 3 확인
- 지속수익률 TOP 3 확인
- 이번 달 참가 종목 확인
- 월간 순위 확인
- 지속 순위 확인
- 누적 참가 종목 검색
- 참가자의 아이디어 메모 확인

할 수 없는 일:

- 데이터 추가
- 데이터 수정
- 데이터 삭제
- 결과 확정
- 시세 수동 갱신

### 관리자

`/admin` 주소로 접근합니다.

관리자 로그인이 필요합니다.

현재 기본 관리자 계정:

- 아이디: `richteacher`
- 비밀번호: `20262026`

관리자는 향후 여러 명 추가될 수 있도록 DB 기반 계정 구조를 사용합니다.

## 4. 대회 규칙

- 참가자 수 제한은 없습니다.
- 한 참가자는 한 기준월에 하나의 참가 종목만 제출할 수 있습니다.
- 한 참가 종목에는 매수 1회만 허용합니다.
- 한 참가 종목에는 매도 1회만 허용합니다.
- 매수일은 기준월 안의 날짜여야 합니다.
- 매도일은 매수일 이후이면서 기준월 안의 날짜여야 합니다.
- 매도하지 않은 경우에는 현재가 또는 결과 확정일 기준 종가를 사용합니다.
- 결과 확정 전 순위는 현재가 기준 임시 순위입니다.
- 결과 확정 후 순위는 확정일 기준 종가로 고정된 공식 순위입니다.
- 지속 순위는 확정된 월간 결과만 누적 계산합니다.

## 5. 핵심 데이터 개념

### participants

참가자 정보입니다.

주요 필드:

- 이름
- 활성 상태
- 삭제 여부

참가자 삭제는 실제 데이터 완전 삭제보다는 비활성/소프트 삭제에 가깝게 다룹니다. 과거 대회 결과 보존이 중요하기 때문입니다.

### contest_months

대회 기준월입니다.

예:

- `2026-06`
- `2026-07`

월별 순위와 참가 종목은 이 기준월을 중심으로 조회됩니다.

### entries

참가 종목 데이터입니다.

주요 필드:

- 참가자
- 기준월
- 종목명
- 종목코드
- 종목코드 없음 여부
- 매수일
- 매수일 종가
- 매도일
- 매도 확정가
- 월말/종료가
- 현재가
- 현재가 기준 수익률
- 아이디어 메모
- 결과 확정 스냅샷

아이디어 메모는 이 서비스에서 중요한 항목입니다. 참가자가 왜 이 종목을 골랐는지 남기는 공간이며, 공개 화면에서도 읽기 좋아야 합니다.

### accounts

관리자 계정입니다.

현재는 관리자 1명만 사용해도 되지만, 나중에 관리자 추가가 가능하도록 DB에 저장하는 구조입니다.

## 6. 순위 계산 방식

### 월간 순위

월간 순위는 기준월의 참가 종목을 대상으로 계산합니다.

결과 확정 전:

- 현재가 기준으로 임시 수익률을 보여줍니다.
- 현재가가 없으면 순위 계산에서 제외될 수 있습니다.

결과 확정 후:

- 확정일 기준 종가를 사용합니다.
- 확정 수익률은 DB에 스냅샷으로 저장됩니다.
- 이후 현재가가 바뀌어도 해당 월의 공식 결과는 변하지 않습니다.

수익률 계산:

```txt
수익률 = (기준 종료가 - 매수가) / 매수가 * 100
```

여기서 기준 종료가는 상황에 따라 다릅니다.

- 결과 미확정: 현재가
- 매도 처리됨: 매도 확정가
- 결과 확정됨: 확정일 기준 종가 또는 확정 스냅샷 값

### 지속 순위

지속 순위는 확정된 월간 결과만 사용합니다.

매월의 확정 수익률을 누적 곱으로 계산해 참가자별 누적 자산지수와 지속수익률을 표시합니다.

예:

```txt
누적자산지수 = 1.0 * (1 + 1월 수익률) * (1 + 2월 수익률) * ...
지속수익률 = (누적자산지수 - 1) * 100
```

미확정 월은 지속 순위에 반영하지 않습니다.

## 7. 시세 조회 구조

### 지원 대상

- 한국 주식
- 한국 ETF
- 미국 주식
- 미국 ETF
- 종목코드가 없는 수동 입력 종목

### 한국 종목

한국 종목은 종목명으로 코드 검색이 가능합니다.

예:

- 삼성전자 -> `005930`
- KODEX 삼성전자단일종목레버리지 -> `0193W0`
- TIGER 미국필라델피아반도체나스닥 -> `381180`

한국 ETF 중 일부는 숫자 6자리 코드가 아니라 영문이 섞인 코드가 있을 수 있습니다. 예: `0193W0`

### 미국 종목

미국 종목은 티커를 입력합니다.

예:

- AAPL
- MSFT
- NVDA
- SPY
- QQQ

미국 티커의 거래소 표시는 데이터 제공처의 한계로 단순화되어 보일 수 있습니다. 예를 들어 다양한 미국 상장 종목이 NASDAQ처럼 표시될 수 있습니다. 실제 가격 조회가 되는지가 더 중요합니다.

### 종목코드 없음

종목코드가 없거나 자동 시세 조회가 어려운 종목은 `종목코드 없음`으로 등록할 수 있어야 합니다.

이 경우:

- 관리자가 가격을 수동 입력합니다.
- 자동 시세 조회 대상에서 제외됩니다.
- 순위 계산은 입력된 가격을 기준으로 진행합니다.

### 잘못된 종목코드

종목코드를 잘못 입력하면 시세 조회가 실패할 수 있습니다.

프론트엔드에서는 다음 상태를 명확하게 보여주는 것이 좋습니다.

- 조회 가능
- 조회 실패
- 코드 없음
- 수동 입력 필요
- 최근 조회일
- 최근 조회가

## 8. 자동 시세 갱신

자동 시세 갱신은 Cloudflare Worker Cron으로 동작합니다.

현재 의도:

- 한국 장 시간 기준으로 평일 09:00-16:45 사이 자동 갱신
- 약 15분 간격 갱신
- 관리자가 매번 버튼을 누르지 않아도 현재가가 갱신되도록 구성

관리자 화면에 수동 `시세 갱신` 기능도 있습니다.

수동 갱신은 다음 상황에서 필요합니다.

- 방금 종목을 등록했을 때
- 자동 갱신 전 즉시 가격을 확인하고 싶을 때
- 조회 실패 후 다시 확인하고 싶을 때

## 9. 공개 화면 구성

공개 화면의 핵심 원칙:

- 첫 화면은 바로 대회 현황이어야 합니다.
- 로그인 없이 순위를 쉽게 확인해야 합니다.
- 관리자 기능은 별도 `/admin`에서만 데이터 변경이 가능해야 합니다.
- 공개 화면에서 관리자 화면으로 이동하는 버튼은 있어야 합니다.

### 상단 영역

상단에는 사이트명과 TOP 3 요약을 표시합니다.

원하는 느낌:

- 과한 장식 없이 간단한 지수 현황판 느낌
- 첫 줄: 서비스명
- 둘째 줄: 월간 TOP 3, 지속 TOP 3
- 월간 TOP 3와 지속 TOP 3는 시각적으로 구분되어야 함

예:

```txt
월간 수익률 대회
월간 TOP3  1위 김OO +12.4%  2위 이OO +9.1%  3위 박OO +7.8%
지속 TOP3  1위 최OO +34.2%  2위 정OO +28.6%  3위 한OO +21.3%
```

### 좌측 사이드 메뉴

데스크톱에서는 좌측 사이드 메뉴를 사용합니다.

메뉴:

- 대시보드
- 현재 시세
- 월간 순위
- 지속 순위
- 참가 종목
- 관리자 모드

### 대시보드

대시보드는 정보를 많이 넣지 않고 가볍게 구성합니다.

표시 항목:

- 이번 달 참가 종목 카드
- 참가하지 않은 사람 이름 목록

이번 달 참가 종목 카드는 작고 간단하게 보여줍니다.

카드에 표시할 정보:

- 참가자 이름
- 종목명
- 현재 또는 확정 수익률

전체 정보는 참가 종목이나 월간 순위 화면에서 확인합니다.

### 월간 순위

기준월별 순위를 보여줍니다.

표시 항목:

- 순위
- 참가자
- 종목명
- 매수일
- 매수가
- 기준 종료일
- 기준 종료가
- 수익률
- 확정 여부

결과 미확정이면 현재가 기준임을 표시해야 합니다.

### 지속 순위

확정된 월간 결과를 누적한 장기 순위입니다.

표시 항목:

- 순위
- 참가자
- 누적자산지수
- 지속수익률
- 반영된 확정 월 수

### 참가 종목

모든 참가 종목을 누적 조회할 수 있는 화면입니다.

필요 기능:

- 기준월 필터
- 참가자 검색
- 종목명 검색
- 종목코드 검색
- 메모 검색
- 확정 여부 필터
- 긴 메모 접기/펼치기
- URL 자동 링크

누적 데이터가 많아지므로 한 화면에 모든 것을 길게 나열하면 안 됩니다.

권장 UI:

- 검색/필터 상단 고정
- 카드 또는 표 형태
- 페이지네이션 또는 더보기
- 모바일에서는 카드형
- 데스크톱에서는 표 또는 compact card

## 10. 관리자 화면 구성

관리자 화면도 공개 화면과 일관된 레이아웃을 유지합니다.

단, 참가 종목 관리는 데이터 입력이 많기 때문에 공개 화면과 똑같은 카드형 디자인을 억지로 맞추면 불편합니다. 관리자 입력 화면은 실무 도구처럼 직관적이고 밀도 있게 구성하는 것이 좋습니다.

### 관리자 사이드 메뉴

메뉴:

- 대회 화면
- 시세 갱신
- 참가자
- 기준월
- 참가 종목
- 관리자 계정
- 로그아웃

`로그아웃`은 다른 메뉴와 조금 분리해서 배치합니다.

### 로그인 화면

DB에 아무 데이터가 없는 최초 상태에서도 관리자 로그인이 가능해야 합니다.

로그인 화면은 `/admin`에서 접근합니다.

### 참가자 관리

기능:

- 참가자 추가
- 참가자 이름 수정
- 참가자 삭제
- 참가자 검색

참가자 수 제한은 없습니다.

### 기준월 관리

기능:

- 기준월 추가
- 현재 기준월 선택
- 기준월별 결과 확정 상태 확인

### 참가 종목 관리

가장 중요한 관리자 화면입니다.

입력 단계는 다음처럼 나누면 좋습니다.

1. 참가자와 종목
2. 매수
3. 매도/종료
4. 아이디어 메모
5. 저장/확정/삭제

#### 1. 참가자와 종목

필드:

- 기준월
- 참가자
- 종목명
- 종목코드
- 종목코드 없음 체크
- 종목코드 검색 버튼
- 종목코드 조회 상태

참가자가 많아질 수 있으므로 탭 방식은 부적합합니다.

권장 UI:

- 참가자 검색 가능한 셀렉트
- 이름 입력 시 필터링
- 최근 선택 참가자 우선 표시

#### 2. 매수

필드:

- 매수일
- 매수일 종가
- 매수일 종가 조회 버튼

매수가는 직접 입력도 가능하지만, 원칙적으로 매수일 종가로 통일합니다.

종목코드가 있으면 `매수일 종가 조회`로 자동 입력됩니다.

#### 3. 매도/종료

필드:

- 매도일 또는 종료일
- 월말/종료가
- 종료가 조회 버튼

현재 정책:

- 매도일을 찍으면 해당일 종가를 조회합니다.
- 월말종가가 없으면 매도 확정가를 월말종가처럼 사용합니다.
- 결과 확정 시에는 확정일 기준 종가를 조회해 최종 순위를 고정합니다.

#### 4. 아이디어 메모

필드:

- 자유 텍스트 메모

요구사항:

- 줄바꿈 보존
- 긴 글 입력 가능
- 공개 화면에서는 긴 글 접기/펼치기
- URL 자동 링크
- HTML 삽입은 허용하지 않음

#### 5. 저장/확정/삭제

기능:

- 참가 종목 저장
- 참가 종목 수정
- 미확정 참가 종목 삭제
- 결과 확정

결과 확정된 참가 종목은 기본 UI에서 쉽게 삭제하거나 수정하지 않는 것이 좋습니다. 과거 결과 보존이 중요하기 때문입니다.

## 11. 모바일 화면 방향

모바일은 데스크톱 기능을 그대로 작게 넣는 방식보다, 핵심 확인과 간단 입력에 집중합니다.

### 공개 모바일

권장 구성:

- 상단 compact header
- TOP 3 요약은 작은 ticker 형태
- 메뉴는 상단 탭 또는 접이식 메뉴
- 대시보드는 2열 카드
- 순위는 카드형 리스트
- 참가 종목은 검색 후 카드형 리스트

### 관리자 모바일

권장 구성:

- 사이드바 대신 상단/하단 compact menu
- 참가 종목 입력은 단계형 폼
- 한 줄에 여러 입력칸을 억지로 넣지 않기
- 날짜, 가격, 조회 버튼은 세로 흐름으로 배치
- 저장/확정/삭제 버튼은 화면 하단에서 찾기 쉽게 배치

## 12. 디자인 방향

원하는 느낌:

- 과한 랜딩페이지 느낌 금지
- 장식적인 카드 남발 금지
- 실전 대회 운영용 도구 느낌
- 작고 명확한 정보 배치
- 지수 현황판처럼 빠르게 읽히는 상단 요약
- 공개 화면은 보기 쉽게
- 관리자 화면은 입력하기 쉽게

피해야 할 것:

- 너무 큰 탑바
- 과한 여백
- 모든 정보를 대시보드에 몰아넣기
- 참가 종목 누적 데이터를 한 화면에 끝없이 스크롤
- 탭으로 많은 참가자 고르기
- 종목 입력란의 열과 행이 어긋나는 레이아웃
- 모바일에서 표가 깨지는 구조

권장 스타일:

- 폰트는 시스템 기본 또는 깔끔한 sans-serif
- 색상은 차분한 배경 + 명확한 상승/하락 색
- 버튼은 기능별로 구분
- 순위와 수익률은 숫자가 잘 보이게
- 입력 폼은 라벨과 필드 정렬을 일정하게
- 데스크톱/모바일 반응형에서 레이아웃이 자연스럽게 변형

## 13. API 개념

실제 구현 세부는 코드 기준으로 확인해야 하지만, 프론트엔드가 이해해야 할 API 개념은 다음과 같습니다.

### 공개 API

```txt
GET /api/months
GET /api/leaderboard?month=YYYY-MM
GET /api/entries?month=YYYY-MM
```

목적:

- 기준월 목록 조회
- 월간/지속 순위 조회
- 참가 종목 조회

### 관리자 API

```txt
POST /api/auth/login
POST /api/auth/logout
POST /api/admin/participants
POST /api/admin/months
POST /api/admin/entries
PATCH /api/admin/entries/:id
POST /api/admin/entries/:id/finalize
DELETE /api/admin/entries/:id
```

목적:

- 로그인/로그아웃
- 참가자 관리
- 기준월 관리
- 참가 종목 생성/수정/삭제
- 결과 확정

### 시세 관련 API

구현상 이름은 코드에서 확인해야 하지만, 기능 개념은 다음과 같습니다.

- 종목코드 검색
- 종목코드 유효성 확인
- 현재가 조회
- 특정일 종가 조회
- 전체 참가 종목 시세 갱신

## 14. 프론트엔드 작업 시 특히 중요한 점

1. 공개 화면과 관리자 화면의 목적을 분리합니다.
   공개 화면은 확인용, 관리자 화면은 입력/수정용입니다.

2. 대시보드는 가볍게 유지합니다.
   TOP 3와 이번 달 참가 종목 요약, 미참가자 정도만 보여줍니다.

3. 참가 종목 누적 조회는 검색과 필터가 핵심입니다.
   데이터가 계속 쌓이므로 무한히 긴 화면이 되면 안 됩니다.

4. 관리자 참가 종목 입력은 가장 사용성이 중요합니다.
   종목코드 조회, 매수일 종가 조회, 종료가 조회, 메모 입력이 빠르게 되어야 합니다.

5. 결과 확정 전과 후를 명확히 구분합니다.
   미확정은 현재가 기준, 확정은 스냅샷 기준입니다.

6. 종목코드 없음과 조회 실패 상태를 분리합니다.
   코드가 없어서 수동 입력하는 경우와, 코드가 틀려서 조회 실패한 경우는 UI에서 다르게 보여야 합니다.

7. 모바일에서는 표보다 카드형이 적합합니다.
   특히 순위와 참가 종목은 모바일 카드 리스트로 보여주는 것이 좋습니다.

8. 과거 결과 보존이 중요합니다.
   확정된 결과를 쉽게 덮어쓰거나 삭제하지 않도록 UX에서 주의해야 합니다.
